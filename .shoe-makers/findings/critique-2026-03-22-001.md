# Critique: Skill Prompt Integration (commits c61b1e8..3e6e66d)

## Summary

The skill prompt integration feature (commit 3e6e66d) connects `generatePrompt()` to the skill registry so work actions include skill-specific instructions. The architecture is sound — clean backward compatibility, correct separation of concerns (skill lookup is pure, no side effects). However, there are mapping bugs that prevent the feature from working for most actions.

## Blocking Issues

### 1. ACTION_TO_SKILL_TYPE mappings don't match actual skill files

The `ACTION_TO_SKILL_TYPE` in `src/prompts.ts` uses these mappings:
- `"fix-tests"` → `"fix-tests"`
- `"write-tests"` → `"test-coverage"`

But the actual skill files have different `maps-to` values:
- `.shoe-makers/skills/fix-tests.md` has `maps-to: fix` (not `fix-tests`)
- `.shoe-makers/skills/test-coverage.md` has `maps-to: test` (not `test-coverage`)

This means `findSkillForAction` will return `undefined` for `fix-tests` and `write-tests` actions — the skill body won't be included in those prompts.

**Fix**: Either update `ACTION_TO_SKILL_TYPE` to use the correct values (`"fix"` and `"test"`), or update the skill files' `maps-to` fields to match.

### 2. `document` action has no skill mapping

The `doc-sync.md` skill exists with `maps-to: doc-sync`, but `ACTION_TO_SKILL_TYPE` has no entry for `"document"`. The document action will never include skill instructions.

**Fix**: Add `"document": "doc-sync"` to `ACTION_TO_SKILL_TYPE`.

## Advisory Issues

### 3. Tests use hardcoded skill bodies, not real skill files

The tests in `prompts.test.ts` create mock skills with `mapsTo: "fix-tests"` and `mapsTo: "test-coverage"`, which match the (incorrect) `ACTION_TO_SKILL_TYPE` mappings. So the tests pass even though the feature is broken in production — the mock skills' `mapsTo` values don't reflect the real skill files.

This is a test quality issue: the tests verify the wiring but not the contract between `ACTION_TO_SKILL_TYPE` and the actual skill files on disk.

### 4. Shift summary integration (separate commit) looks correct

The `shift.ts` changes adding `summary: ShiftSummary` to `ShiftResult` and the `shift-summary.test.ts` type fixes are clean and correct.

### 5. schedule.ts exclusion in invariants.ts is correct

Adding `schedule.ts` to `EXCLUDED_TOP_LEVEL` prevents it from being flagged as unspecified code.

## Verdict

The skill prompt integration architecture is correct but the implementation has mapping bugs that prevent it from working for 3 out of 5 mapped actions. This should be fixed before proceeding with new work.

## Status

Resolved.
