The `.shoe-makers/state/` directory is gitignored, which means `last-reviewed-commit` won't persist across clones or branches. The gatekeeping system relies on this file existing. If it doesn't exist, `checkUnreviewedCommits` returns `false` (no unreviewed commits), which silently skips review. This should either be un-gitignored or handled differently.

The resolution check regex is too loose — it matches if "Status" and the word meaning "done/fixed" appear anywhere in the file text, even in discussion about the pattern. A dedicated status line at the end would be more robust.
