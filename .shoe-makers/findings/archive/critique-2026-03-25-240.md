### 1. Archive overwrite — data loss (324c5b7) — BUG

The housekeeping auto-commit `324c5b7` archived `critique-2026-03-25-002.md` from the findings directory into the archive, but the archive already contained a different file with the same name. The original archived critique (a doc-sync review of commit `7bd88d6` about stale action types in wiki pages) was **overwritten** with the newer typecheck critique. This is data loss — the original doc-sync critique content is only recoverable from git history.

Root cause: the archive process uses the same filename when moving a resolved critique, and if that filename already exists in the archive (because it was previously archived under a different name and then a new critique reused the sequence number), the old content is silently replaced. This is a systemic risk — any time a critique file is created with a sequence number that was previously used and archived, the archive will be corrupted on the next archive cycle.
