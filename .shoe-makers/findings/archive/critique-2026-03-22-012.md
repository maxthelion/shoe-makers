# Critique: Fix failing checkUnreviewedCommits test

## Commit reviewed

- `f006466` — Fix failing checkUnreviewedCommits test: use HEAD~1 instead of merge-base

## Summary

The previous elf was tasked with fixing a failing test. The test `checkUnreviewedCommits > returns true when commits exist after marker` failed because it used `git merge-base HEAD origin/main` as the marker, but on this branch HEAD equals the merge-base, so `git log marker..HEAD` returned nothing.

The fix changes the marker to `HEAD~1`, which reliably has at least one commit after it (HEAD itself).

## Assessment

**Correct fix.** The change is minimal, targeted, and addresses the root cause. The test now uses a marker (`HEAD~1`) that is guaranteed to be behind HEAD by at least one commit.

### Compliance with rules

- Source/test modification: Modified a test file, which is permitted when the test itself is incorrect. The test had a flawed assumption (merge-base != HEAD), so fixing the test is appropriate. ✓
- No tests deleted or skipped. ✓
- No unrelated files modified. ✓
- `bun test` passes (299/299). ✓

### Potential concern

- **Edge case**: `HEAD~1` assumes the branch has at least 2 commits. On a brand-new branch with only one commit, `git rev-parse HEAD~1` would fail. However, this test runs against the real repo which always has history, so this is acceptable in practice. The original `merge-base` approach had the same fragility (requires an `origin/main` remote).
