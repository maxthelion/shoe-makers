## Summary

Added a JSDoc drift test to `src/__tests__/default-tree.test.ts` that verifies the JSDoc comment has the same number of tree branch entries (├── or └──) as the actual tree has children. Tests: 498 → 499.

## Note

The test verifies count, not content. If someone adds a tree node but updates the JSDoc with a wrong description, the test would still pass. A stronger test would verify the mapping between node names and JSDoc descriptions (e.g. "tests-failing" → "tests failing"). However, this requires a naming convention or explicit mapping table, which adds complexity. The current count-based check is a reasonable first step that catches the most common drift (adding/removing nodes without updating JSDoc).
