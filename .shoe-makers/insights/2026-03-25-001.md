# Insight: Quorum Sensing for Shift Progress

## Lens

**Quorum Sensing** — bacteria communicate through chemical signalling molecules to coordinate group behaviour. When enough bacteria are present (a quorum), they collectively switch on genes for bioluminescence, biofilm formation, or virulence — behaviours that only work at scale.

## Connection

The review-loop-breaker is a binary circuit breaker: either the loop count is below 3 (normal) or above (break out). But quorum sensing is graduated — bacteria accumulate signal molecules over time, and different concentration thresholds trigger different responses.

The shoe-makers system could benefit from a similar graduated response. Instead of a binary breaker that forces explore (which can't progress to prioritise when candidates already exist), the system could:

1. **Low signal (1-2 review loops)**: Continue normally — some review cycling is healthy.
2. **Medium signal (3-4 loops)**: Force explore but allow progression to prioritise if candidates exist — the current breaker blocks this.
3. **High signal (5+ loops)**: Force explore AND clear candidates — something structural is wrong, fresh exploration needed.

## Specific Proposal

The `review-loop-breaker` node currently routes to `explore` unconditionally. When `candidates.md` already exists (meaning exploration was already done), this creates a stuck state where the tree keeps routing to explore even though the work is done. The breaker should check `hasCandidates` — if candidates exist, route to `prioritise` instead of `explore`.

This is the "concentration threshold" idea from quorum sensing: the same signal (review loops) triggers different responses based on the current state (candidates exist or not).

## Impact

This would prevent wasted ticks where the system re-explores when exploration is already complete. Currently the breaker fires every tick for the rest of the shift, burning through the max-ticks-per-shift budget on redundant exploration.
