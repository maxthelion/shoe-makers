## Lens

**Kayapinar, Kiziltepe** — A small Kurdish neighbourhood with dual naming (official Turkish vs. local Kurdish) and a single-tribe identity within a larger hierarchy. Used as a lens for informal knowledge transfer between elf invocations.

## Connection

Each elf invocation produces formal artifacts (commits, findings, work-items) but loses informal context — things noticed, dead ends explored, patterns observed. The "official name" (commit message, work-item title) survives; the "local name" (what the elf actually experienced) dies.

## Proposal (reworked)

Rather than a new state file, leverage the existing findings mechanism with a lightweight "note" type. Notes are one-paragraph ephemeral observations that auto-archive after 24 hours. The setup script would include the text of active notes in the next elf's prompt context, giving each invocation informal tribal knowledge from previous elves in the same shift.

## Evaluation

The core insight — that informal context is lost between invocations — is real. Three existing mechanisms partially address it (`known-issues.md`, findings, shift log), but none capture **within-shift ephemeral observations** that don't warrant a formal finding.

The original proposal (new state file) has a practical problem: `.shoe-makers/state/` is scheduler-managed, and a new file type would require setup script changes to be useful.

The better version: add a "note" variant to the existing findings system. Findings already have a directory, an archive mechanism, and are counted by the setup script. Adding note-type findings that auto-archive is a small delta on existing infrastructure.

## Build On It

Implementation:
1. Convention: note findings use filename pattern `note-{YYYY-MM-DD}-{NNN}.md` with a `## Status: Note` section
2. `src/setup.ts` — when building prompt context, read any active note-type findings and include their text (just the first paragraph)
3. Setup's archive logic — auto-archive notes older than 24 hours
4. Update skill prompts to mention that elves can write quick notes for ephemeral observations

This reuses existing infrastructure without adding new state file types. The key insight from the Kayapinar lens survives: give the "local name" (informal experience) a place to live alongside the "official name" (formal artifacts).

## Decision

**Rework** — the improved version (note-type findings) is concrete and low-risk, but needs a future evaluator or prioritiser to decide if the implementation complexity is worth it given the current tier priorities.
