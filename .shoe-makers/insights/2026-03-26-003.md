## Lens

**Lemmer (whaling)** — A lemmer is a specialized worker in the whale processing pipeline. After the flenser strips blubber, the lemmer dismembers the carcass — separating meat from bones so it can be boiled for oil. The key insight: whaling was a pipeline of specialized roles (harpooner → flenser → lemmer), each receiving the output of the previous stage in a progressively more refined form.

## Connection

The shoe-makers three-phase cycle (explore → prioritise → execute) mirrors this pipeline structure. The explore elf is the harpooner — casting wide, finding candidates. The prioritise elf is the flenser — stripping away the low-value options, exposing the core work item. The execute elf is the lemmer — doing the detailed separation work on a single, well-prepared item.

But the analogy reveals a gap: in whaling, the lemmer worked on a **lemmer deck** — a dedicated workspace where the carcass was pulled for processing. The execute elf has no equivalent staging area. It reads `work-item.md` and then must figure out which files to read, which patterns to follow, and what the current state of the relevant code is. The prioritise elf writes instructions, but doesn't prepare the "deck."

## Proposal

Enhance the prioritise phase to create a **context bundle** alongside the work-item — a `.shoe-makers/state/work-context/` directory containing symlinks or excerpts of the specific files the executor will need. Instead of the work-item saying "read src/setup.ts and src/skills/assess.ts", the prioritiser would extract the relevant sections and include them directly in the work-item prompt.

Concretely: the `buildPrioritisePrompt` could instruct the prioritise elf to include inline code excerpts in `work-item.md` itself — the relevant function bodies, the test patterns to follow, the wiki sections that describe the spec. This "lemmer deck" preparation means the execute elf doesn't need to do its own exploration before starting work.

## Why

Currently, the execute elf often spends its first minutes re-reading files that the prioritise elf already read. This is wasted work — the prioritise elf had all the context but only passed along high-level instructions. By preparing a richer work-item with inline code context (the "lemmer deck"), the execute elf can start cutting immediately. The whaling pipeline's efficiency came from each stage preparing the material for the next — the shoe-makers pipeline should do the same.
