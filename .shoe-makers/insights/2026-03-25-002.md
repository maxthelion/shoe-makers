# Insight: Commit Message as Structured Data (Reworked)

## Lens

**Unknown** — Observation that commit messages serve dual roles: human-readable narrative AND machine-parseable state signals.

## Connection

The shoe-makers system uses commit messages as state signals (`HOUSEKEEPING_PREFIX` in `detect-violations.ts`). The insight proposes git trailers to formalize this.

## Evaluation (generous disposition)

The core idea — formalizing commit metadata — is sound. But the original proposal has a practical obstacle: the elf writes commits, not the setup script. There's no enforcement mechanism to ensure elves include the right trailers, and LLMs don't reliably follow formatting constraints.

### Improved variant: post-commit trailer injection

Instead of asking the elf to write trailers, the setup script could:
1. Write `.shoe-makers/state/current-action-type` before the elf starts
2. After the elf commits, a git post-commit hook (or the next setup run) reads the action type and amends the commit to add trailers

This keeps the elf's job simple (just write good commit messages) while adding structured metadata automatically.

### Even simpler first step: action-type prefix convention

The elf already writes commit messages like "Explore: write candidates..." and "Adversarial review: critique (clean)..." — these naturally contain the action type. Rather than trailers, standardize the prefix convention and parse it:

```
<Action-Type>: <description>
```

This is already ~80% adopted. A simple regex could extract the action type from any commit in the range, replacing both `HOUSEKEEPING_PREFIX` string matching and the state-file path checking with a single, consistent pattern.

## Proposal (revised)

1. Standardize commit message prefixes in the prompt: `Fix-tests:`, `Critique:`, `Explore:`, `Prioritise:`, `Execute:`, `Innovate:`, `Evaluate-insight:`, `Setup-housekeeping:`
2. Parse the prefix in `getElfChangedFiles` to classify commits by type
3. Only require full review for commits with source-code-modifying prefixes (`Fix-tests:`, `Execute:`, `Fix-critique:`)

## Why

- Simpler than trailers — works with what elves already do
- No enforcement mechanism needed — the prefix is already in the prompt
- More robust than file-path checking — a commit that moves a finding to archive shouldn't trigger review just because it's not under `.shoe-makers/state/`
