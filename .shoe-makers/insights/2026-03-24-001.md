# Insight: Graceful Degradation for Creative Pipeline

## Lens: *Graceful Degradation in Distributed Systems* — When external dependencies fail, systems should fall back to reduced but still functional behaviour rather than failing entirely.

(Note: Wikipedia article fetch failed this tick due to network restrictions. This insight is itself an example of the problem it describes.)

## Connection

The shoe-makers creative pipeline currently has a hard dependency on Wikipedia's API for the innovate action. When the fetch fails, the elf receives "Unknown / No article fetched" but is still told "MUST use the Wikipedia article provided above as the lens." This creates an impossible task — the elf is required to reference an article that doesn't exist.

This is exactly the graceful degradation problem: an external dependency (Wikipedia API) is unavailable, but the system doesn't adapt its instructions.

## Proposal

Add a fallback creative source when Wikipedia is unavailable:

1. **Curated seed concepts**: Ship a small file `.shoe-makers/creative-seeds.md` with 20-30 diverse concepts (from domains like biology, music, architecture, game design, economics). When Wikipedia fails, randomly pick one.

2. **Prompt adaptation**: When no article is fetched, the innovate prompt should explicitly say "Use this seed concept as your lens" rather than "Use the Wikipedia article above" — don't give instructions that reference missing content.

3. **Previous insights as lenses**: If seed concepts are exhausted, use a previous insight's Lens section as the starting point for a new, deeper exploration.

This ensures the creative pipeline always produces output, even offline.

## Why

- The current behaviour creates a contradiction: mandatory output + missing input = confused elf
- Offline/restricted environments (like this one) completely block the innovation tier
- Curated seeds could actually produce *better* insights than random Wikipedia articles, since they'd be pre-selected for interesting cross-domain potential
- The fallback chain (Wikipedia → seeds → previous insights) makes the system antifragile
