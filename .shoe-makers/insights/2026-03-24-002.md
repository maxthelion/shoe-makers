# Insight: Mycorrhizal Network — Shared Memory Between Elves

## Lens: *Mycorrhizal Networks from Forest Ecology* — Underground fungal networks connect trees, allowing them to share nutrients, water, and chemical signals. Older "mother trees" redistribute resources to struggling seedlings. The network has no central coordinator — it emerges from local connections.

## Connection

Each shoe-maker elf currently starts fresh with no memory of what previous elves learned. The shift log records what happened, but there's no structured way for one elf's discoveries to benefit future elves. For example:

- An explore elf discovers that `src/verify/invariants.ts` has a subtle bug in claim ID matching — it notes this in a finding, but the next explore elf starts from scratch and may not even read findings
- A critique elf learns that evidence patterns with markdown bold (`**text**`) don't match — this knowledge dies with the elf
- An innovate elf discovers a great cross-domain connection but only writes one insight — the understanding of *why* that connection worked isn't captured

In mycorrhizal networks, nutrients flow through existing connections — not de novo each time. The shoe-makers system lacks this "underground network."

## Proposal

Add a `.shoe-makers/knowledge/` directory for **persistent elf knowledge** — structured, grep-able observations that accumulate across shifts:

```markdown
# .shoe-makers/knowledge/evidence-patterns.md
## Lessons Learned
- Evidence patterns are matched against code with comments stripped, but markdown formatting (**, *, etc.) in template literals is NOT stripped
- Claim IDs are generated by slugifying text to 60 chars — verify exact IDs with the pipeline, don't guess
- The `###` regex only matches two-level numbering (2.6), not three-level (2.6.1)
```

Each skill prompt would include relevant knowledge snippets (grep for the skill type or domain). Unlike findings (which are problems to fix) or insights (which are creative proposals), knowledge entries are **facts that help future elves work better**.

## Why

- Reduces reactive churn — elves don't rediscover the same gotchas
- Knowledge accumulates across shifts instead of resetting
- Low implementation cost — just a directory convention + a grep in the prompt generator
- Mirrors the mycorrhizal model: local discoveries become shared infrastructure, no central coordinator needed
