# Add test for loadSkills with empty enabledSkills array

skill-type: test-coverage

## What needs to change

Add one focused test to `src/__tests__/registry.test.ts` for the edge case where `enabledSkills` is an empty array `[]`. Currently tested: null (all), specific list (filters). Untested: empty array (should load zero skills).

### Test to add

Inside the existing `describe("loadSkills", ...)` block (around line 161), add:

```typescript
test("loads no skills when enabledSkills is empty array", async () => {
  tmpDir = await mkdtemp(join(tmpdir(), "registry-test-"));
  const skillsDir = join(tmpDir, ".shoe-makers", "skills");
  await mkdir(skillsDir, { recursive: true });

  await writeFile(
    join(skillsDir, "fix-tests.md"),
    `---\nname: fix-tests\ndescription: Fix tests.\nmaps-to: fix\nrisk: low\n---\nFix.`,
  );

  const skills = await loadSkills(tmpDir, []);
  expect(skills.size).toBe(0);

  await rm(tmpDir, { recursive: true });
});
```

## What NOT to change

- Do not modify source files — only add a test
- Do not modify existing tests
- Do not modify .shoe-makers/invariants.md

## Decision Rationale

Candidate #1 (registry filtering) was chosen because the empty-array edge case is a genuine gap — it tests the boundary between "filter to specific skills" and "filter to nothing". Candidates #2 and #3 were already covered by existing tests/docs upon closer inspection.
