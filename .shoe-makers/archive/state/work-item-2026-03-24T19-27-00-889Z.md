# Work Item: Add tests for prompt helper pure functions

skill-type: test

## Context

`src/prompts/helpers.ts` contains several pure functions that control tier routing, gap formatting, skill catalog rendering, and action type parsing. None have direct unit tests. These functions are used by the behaviour tree and prompt system.

## What to do

Create `src/__tests__/prompt-helpers.test.ts` with tests for:

1. **`determineTier()`**: test with no assessment, zero gaps, specOnly > 0, untested >= 5, both gaps
2. **`isInnovationTier()`**: test returns false when no assessment, false when gaps exist, true when no gaps
3. **`formatTopGaps()`**: test empty gaps returns empty string, formats up to 5 gaps
4. **`formatCodebaseSnapshot()`**: test with null assessment, with health/worst files/findings
5. **`parseActionTypeFromPrompt()`**: test known titles map correctly, unknown title returns null
6. **`findSkillForAction()`**: test with no skills map, with matching skill, with non-matching action
7. **`formatSkillCatalog()`**: test empty/absent skills, formatted output

## Patterns to follow

Follow existing test files like `src/__tests__/action-constants.test.ts` — use `describe/test/expect` from `bun:test`, group by function name.

## What NOT to change

- Do not modify `src/prompts/helpers.ts` — only add tests
- Do not modify other test files

## Decision Rationale

These are pure functions with clear inputs/outputs — ideal for unit testing. They control critical system behavior (tier routing, permission parsing) and are currently unverified.
