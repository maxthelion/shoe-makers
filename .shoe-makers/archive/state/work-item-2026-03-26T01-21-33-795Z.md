# Implement Local Creative Corpus Reader

## Wiki Spec

From `.shoe-makers/invariants.md` (specified-only claims):
- "Random concepts for the creative lens come from `.shoe-makers/creative-corpus/` — markdown files with title, source URL, and a summary"
- "The corpus is populated locally by running `scripts/fetch-wikipedia-corpus.sh`"
- "Setup picks one unused article at random from the corpus and embeds it in the innovate prompt"

The invariants also describe: "After the innovate tick completes, the article is marked as used by adding `used: true` to its frontmatter — each article is used only once."

## Current Code

`src/creative/wikipedia.ts` — Contains:
- Lines 5-77: `FALLBACK_CONCEPTS` hardcoded array (50+ concepts)
- Lines 82-84: `getRandomFallbackConcept()` picks from hardcoded array
- Lines 90-123: `fetchRandomArticle()` tries Wikipedia API, falls back to hardcoded array
- Lines 139-161: `fetchArticleForAction()` orchestrates article fetching for setup

The local corpus at `.shoe-makers/creative-corpus/` contains 20 markdown files with frontmatter:
```yaml
---
title: "Article Title"
source: https://en.wikipedia.org/wiki/...
fetched: 2026-03-25T10:46:01Z
---
Article summary text...
```

**The gap:** `fetchRandomArticle()` never reads from the local corpus. It tries Wikipedia API, then falls back to the hardcoded `FALLBACK_CONCEPTS`. The 20 local corpus files are never used.

## What to Build

1. **Add `readLocalCorpus()` function** in `src/creative/wikipedia.ts`:
   - Read all `.md` files from `.shoe-makers/creative-corpus/`
   - Parse frontmatter (title, source, fetched, used)
   - Filter to unused articles (those without `used: true` in frontmatter)
   - Return array of `{ title, summary, filepath }`

2. **Add `pickFromCorpus()` function** in `src/creative/wikipedia.ts`:
   - Call `readLocalCorpus()` to get unused articles
   - Pick one at random
   - Return `{ title, summary }` or null if corpus is empty/all used

3. **Add `markArticleUsed()` function** in `src/creative/wikipedia.ts`:
   - Takes a filepath
   - Reads the file, adds `used: true` to frontmatter
   - Writes back

4. **Update `fetchRandomArticle()`** to use this priority:
   1. Try Wikipedia API (existing code, lines 96-119)
   2. If API fails, try local corpus via `pickFromCorpus()`
   3. If corpus exhausted, fall back to `getRandomFallbackConcept()` (existing)

5. **Update `fetchArticleForAction()`** to mark the article as used after the innovate action:
   - Return additional metadata (the filepath) so setup can mark it used after the tick
   - OR: mark it used immediately when picked (simpler, acceptable since the innovate prompt will include it)

## Patterns to Follow

- Parse frontmatter using simple regex (like `src/setup/format-action.ts:61` strips frontmatter). Don't add a YAML parser dependency.
- Use `readdir` + `readFile` (already used throughout the codebase, e.g. `src/verify/parse-evidence.ts:110`)
- Keep the function pure where possible — `readLocalCorpus` is pure (reads files), `markArticleUsed` has a side effect (writes file)
