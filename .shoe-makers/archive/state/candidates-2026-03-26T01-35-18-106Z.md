# Candidates

## 1. Implement structured explore skill — pre-fill output template with codebase metrics
**Type**: implement
**Impact**: high
**Reasoning**: Per `wiki/pages/structured-skills.md` lines 74-77, the `write-candidates` (explore) skill should enforce minimum 3 candidates with required fields (type, impact, reasoning) via a pre-filled output template. Currently `src/prompts/explore.ts` passes metrics as prose. Following the structured critique pattern (`src/setup/critique-context.ts`), create `ExploreContext` that gathers invariant counts, health scores, worst files, findings count, and tier, then builds a `[YOUR JUDGEMENT HERE]` template for the elf to fill in. This reduces format compliance failures in explore outputs.

## 2. Implement structured prioritise skill — pre-fill candidates and work-item template
**Type**: implement
**Impact**: medium
**Reasoning**: Per `wiki/pages/structured-skills.md` lines 79-82, the `write-work-item` (prioritise) skill should enforce required sections (title, skill-type, wiki text, instructions, tests, off-limits, decision rationale). ## 3. Implement structured innovate skill — enforce insight output format
**Type**: implement
