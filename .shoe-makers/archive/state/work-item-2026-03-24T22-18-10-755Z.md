# Add missing `innovate` and `evaluate-insight` cases to run-skill.ts

skill-type: bug-fix

## Context

The `ActionType` union in `src/types.ts` (lines 27-38) defines 11 action types:
```
fix-tests, fix-critique, critique, review, inbox, execute-work-item, dead-code, prioritise, innovate, evaluate-insight, explore
```

The `runSkill()` function in `src/scheduler/run-skill.ts` (lines 15-47) has a switch statement that handles 9 of these but is missing `innovate` and `evaluate-insight`. These fall through to `default`, returning "Unknown action: innovate" (or "evaluate-insight").

## What to build

Add two cases to the switch in `src/scheduler/run-skill.ts`:

```typescript
case "innovate":
  return "Action: innovate — elf should write a creative insight from the innovation brief.";

case "evaluate-insight":
  return "Action: evaluate-insight — elf should read an insight and evaluate whether to act on it.";
```

Place them before the `default` case, after `prioritise` (line 43).

## Pattern to follow

Look at the existing cases (lines 21-43). Each returns a one-line string with format: `"Action: {type} — {description}."` The descriptions should match what the tree/prompt system expects the elf to do for that action.

## Tests to write

In `src/__tests__/run-skill.test.ts`, add two tests following the existing pattern:

```typescript
test("returns descriptive message for innovate", async () => {
  const result = await runSkill(tempDir, "innovate");
  expect(result).toContain("innovate");
});

test("returns descriptive message for evaluate-insight", async () => {
  const result = await runSkill(tempDir, "evaluate-insight");
  expect(result).toContain("evaluate-insight");
});
```

## What NOT to change

- Do not modify `src/types.ts`
- Do not modify the `explore` case (it runs programmatic assessment)
- Do not modify the shift runner (`src/scheduler/shift.ts`)
- Do not modify any prompt files

## Decision Rationale

Candidate #1 was chosen over #2 (fs.ts tests, low impact) and #3 (README accuracy, requires reading and evaluating prose) because it's a concrete code bug with a clear fix. The missing switch cases cause incorrect log output ("Unknown action") whenever the shift runner encounters `innovate` or `evaluate-insight`, which happens 3+ times per shift based on today's log. Fixing this improves observability and prevents confusion when debugging shift behavior.
