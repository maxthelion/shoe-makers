# Work Item: Add shift-log-parser tests

skill-type: test

## What to test

`src/log/shift-log-parser.ts` exports three functions:
- `parseShiftLogActions(logContent)` — parses action names from shift log entries
- `computeProcessPatterns(actions)` — computes reactive ratio, review loop count, innovation cycle count
- `getShiftProcessPatterns(repoRoot)` — reads today's log and combines the above

## Tests to write in `src/__tests__/shift-log-parser.test.ts`

### parseShiftLogActions
- Parses a log with multiple action entries
- Returns actions in order
- Handles unrecognized action titles (skips them)
- Returns empty array for empty log
- Handles log with no action lines

### computeProcessPatterns
- Reactive ratio: all reactive → 1.0
- Reactive ratio: all proactive → 0.0
- Reactive ratio: mixed → correct fraction
- Reactive ratio: no actions → 0
- Review loop count: 3+ consecutive critique/fix-critique → count 1
- Review loop count: less than 3 consecutive → count 0
- Review loop count: multiple loops → correct count
- Innovation cycle count: counts only "innovate" actions

### getShiftProcessPatterns
- Returns undefined when no shift log exists

## Decision Rationale
Chosen as highest impact: the parser drives tree routing (innovation tier gate via innovationCycleCount) and prompt signals (reactive ratio in explore prompt). Zero tests means silent regressions could produce wrong routing decisions.
