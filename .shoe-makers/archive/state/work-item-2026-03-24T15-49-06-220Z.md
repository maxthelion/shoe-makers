# Add .shoe-makers/state/ to HOUSEKEEPING_PATHS

skill-type: implement

## Problem

The auto-commit housekeeping mechanism in setup.ts doesn't recognize `.shoe-makers/state/` as a housekeeping path. When consumed state files (work-item.md, candidates.md) are deleted by setup's archive mechanism, they're left as uncommitted changes that trigger the `unverified-work` tree condition, creating a review→housekeeping tight loop.

## Fix

In `src/setup.ts` line 163, add `.shoe-makers/state/` to the `HOUSEKEEPING_PATHS` array:

```typescript
export const HOUSEKEEPING_PATHS = [
  ".shoe-makers/findings/",
  ".shoe-makers/log/",
  ".shoe-makers/archive/",
  ".shoe-makers/state/",
];
```

## Tests

Add a test to `src/__tests__/auto-commit-housekeeping.test.ts` verifying that `.shoe-makers/state/work-item.md` deletion is recognized as housekeeping by `isAllHousekeeping()`.

## What NOT to change
- `.shoe-makers/invariants.md`
- The auto-commit logic itself — just the path list
