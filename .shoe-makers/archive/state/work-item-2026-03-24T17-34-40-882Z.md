# Add roundtrip test for init skill templates

skill-type: test-coverage

## Problem

The existing `init-templates.test.ts` verifies skill templates have the right structure (frontmatter fields, section headers), but doesn't verify the templates can actually be parsed by `parseSkillFile()`. If a template has a subtle format issue (e.g., extra whitespace in frontmatter), the init would succeed but the registry would fail to load the skill.

## What to do

1. Add a test in `src/__tests__/init-templates.test.ts` that calls `parseSkillFile()` from the registry for each skill template
2. Verify the parsed result has the expected name, non-empty description, valid risk level, and non-empty body
3. This is a roundtrip test: template → parseSkillFile → validate fields

## What NOT to change

- Do not modify the templates themselves
- Do not modify `parseSkillFile()` or the registry
- Do not modify `.shoe-makers/invariants.md`

## Decision Rationale

This is the most impactful remaining candidate — it catches a class of errors (template format regressions) that no existing test covers. The roundtrip pattern ensures the templates stay in sync with the registry's expectations.
