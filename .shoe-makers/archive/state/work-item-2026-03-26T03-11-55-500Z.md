# Implement note-type findings for ephemeral elf context

## Background

Creative insight from Kayapinar lens: informal context is lost between elf invocations. The existing findings mechanism can be extended with a lightweight "note" type — ephemeral one-paragraph observations that auto-archive after 24 hours.

## What to build

### 1. Note file convention

Notes use filename pattern `note-{YYYY-MM-DD}-{NNN}.md` in `.shoe-makers/findings/`. Content is a single markdown file with:

```markdown
# Note

[One or two sentences of informal context for the next elf.]
```

No frontmatter, no status section — just a title and a paragraph. Keep it minimal.

### 2. Setup script: include active notes in prompt context

In `src/setup.ts`, after reading findings, also read any `note-*.md` files and include their text in the action prompt context. Add a section like:

```
## Notes from previous elves

- [text of note 1]
- [text of note 2]
```

Only include notes from the current date (today's notes). This keeps context fresh and relevant.

### 3. Setup script: auto-archive old notes

In the setup script's archive logic, auto-archive any `note-*.md` files where the date in the filename is older than 24 hours. This prevents note accumulation.

### 4. Update action prompts

Add a line to action prompts (in the "After completing" section or similar) mentioning that elves can write a quick note file if they discover something informally useful for the next elf.

## Files to modify

- `src/setup.ts` — read notes, include in prompt, auto-archive old notes
- Skill/prompt templates that generate action prompts — add note-writing guidance

## Tests

- Test that notes with today's date are read and included in prompt context
- Test that notes with yesterday's date are auto-archived
- Test that note content is correctly extracted (just the paragraph, not the heading)

## Patterns to follow

- Follow existing findings reading pattern in setup.ts
- Follow existing archive logic for moving files to `findings/archive/`
