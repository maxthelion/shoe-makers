# Candidates

## 1. Add shift-log-parser tests
**Type**: test
**Impact**: high
**Reasoning**: `src/log/shift-log-parser.ts` provides `getShiftProcessPatterns()` which drives the reactive ratio, review loop count, and innovation cycle count. These values affect tree routing (innovation tier gate) and explore/prioritise prompt signals. The parser has zero dedicated tests — any parsing regression would silently produce wrong signals. This is the most impactful untested code path.
**Files**: `src/log/shift-log-parser.ts` (source), `src/__tests__/shift-log-parser.test.ts` (new)

## 2. Consolidate autoCommitHousekeeping calls in setup.ts
**Type**: improve
**Impact**: medium
**Reasoning**: `autoCommitHousekeeping` in `src/setup.ts` is called twice (line 58 and line 168), creating up to two extra commits per setup run. Each commit can trigger the `unreviewed-commits` condition in the next tick. Consolidating into a single call at the end of setup would halve the housekeeping commit count. Now that the permission violation false positive is fixed, this is less urgent but still reduces unnecessary review cycles.
**Files**: `src/setup.ts`

## 3. Add SETUP_HOUSEKEEPING_PATHS to isAllHousekeeping for consistency
**Type**: improve
**Impact**: medium
**Reasoning**: `src/setup.ts` defines `HOUSEKEEPING_PATHS` (which includes `.shoe-makers/state/`) and `src/utils/housekeeping.ts` defines `SETUP_HOUSEKEEPING_PATHS` (without state). Both are used for housekeeping detection but with different semantics. The `isAllHousekeeping()` function in setup.ts should be refactored to use the shared constant for consistency and to avoid drift between the two definitions.
**Files**: `src/setup.ts`, `src/utils/housekeeping.ts`

## 4. Add reactive loop detector to process patterns
**Type**: implement
**Impact**: medium
**Reasoning**: The process patterns track reactive ratio but don't detect consecutive same-action firing (e.g., "unresolved-critiques" firing 5 times in a row). A "consecutive same-action count" would help explore elves diagnose stuck loops. This would require parsing the shift log for action sequences.
**Files**: `src/log/shift-log-parser.ts`, `src/types.ts`, `src/prompts/three-phase.ts`
