# Add tests for reactive.ts and three-phase.ts prompt generation

skill-type: test-coverage

## What needs to change

`src/prompts/reactive.ts` and `src/prompts/three-phase.ts` export functions that build the prompts elves receive. While `src/__tests__/prompts.test.ts` tests `generatePrompt()` (the index entry point), these two modules should be tested directly to cover edge cases.

### reactive.ts (src/prompts/reactive.ts)

Key functions and what to test:
- `buildFixTestsPrompt(skillSection)` — receives a skill section string, should include it in output
- `buildFixCritiquePrompt()` — static prompt, verify structure
- `buildCritiquePrompt(permissionViolations?)` — test with and without violations, verify violation list appears in output when provided
- `buildReviewPrompt()` — static prompt, verify structure
- `buildInboxPrompt(state)` — verify inbox count appears in output

### three-phase.ts (src/prompts/three-phase.ts)

Key functions and what to test:
- `buildExplorePrompt(state, skills?, article?)` — test with and without article lens, test tier detection affects output
- `buildPrioritisePrompt(state)` — verify tier affects guidance text
- `buildExecutePrompt(skillSection)` — receives skill section, verify it's included
- `buildInnovatePrompt(state, article?, wikiSummary?)` — test with/without article, with/without wiki summary
- `buildEvaluateInsightPrompt(state)` — verify insight count appears
- `buildDeadCodePrompt(skillSection)` — verify structure
- `formatProcessTemperature(assessment)` — test high reactive ratio (>0.6), low (<0.3), and moderate (between)

### Test file

Add tests to a new file `src/__tests__/prompt-builders.test.ts`. Follow the pattern in `src/__tests__/prompts.test.ts`:
- Import from `../prompts/reactive` and `../prompts/three-phase` directly
- Use `emptyBlackboard` from `./test-utils` for WorldState
- Verify output contains expected strings (not exact match — prompts are prose)

### Example test patterns

```typescript
test("buildCritiquePrompt includes permission violations", () => {
  const result = buildCritiquePrompt(["src/foo.ts", "wiki/bar.md"]);
  expect(result).toContain("PERMISSION VIOLATIONS");
  expect(result).toContain("src/foo.ts");
  expect(result).toContain("wiki/bar.md");
});

test("buildCritiquePrompt without violations has no warning", () => {
  const result = buildCritiquePrompt();
  expect(result).not.toContain("PERMISSION VIOLATIONS");
});

```

## What NOT to change

## Decision Rationale

Candidate #1 (README config) was already completed. Of the remaining test candidates, prompt generation tests have the highest impact — prompts are the primary interface between the tree and the agents, and formatting bugs here would affect all elf behaviour. The other test candidates (parser boundaries, registry filtering) cover lower-risk utilities.
