# Candidates

## 1. Implement local creative corpus reader — use .shoe-makers/creative-corpus/ instead of hardcoded fallback
**Type**: implement
**Impact**: high
**Reasoning**: The wiki spec and invariants.md claim that "Setup picks one unused article at random from the corpus" (`.shoe-makers/creative-corpus/` markdown files) and marks articles as `used: true` in frontmatter. Currently `src/creative/wikipedia.ts` lines 5-77 use a hardcoded `FALLBACK_CONCEPTS` array instead. The 20 markdown files in `.shoe-makers/creative-corpus/` are never read. Implementation: add a `readLocalCorpus()` function that reads the corpus directory, filters for unused articles (no `used: true` in frontmatter), picks one at random, and after the innovate tick marks it as used. This closes 3 specified-only invariant gaps related to creative corpus management.
