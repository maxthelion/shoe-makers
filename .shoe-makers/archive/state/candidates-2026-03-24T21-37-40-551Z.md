# Candidates

## 1. Remove dead code modules: verify.ts, prioritise.ts, work.ts
**Type**: dead-code
**Impact**: high
**Reasoning**: `src/skills/verify.ts`, `src/skills/prioritise.ts`, and `src/skills/work.ts` exist and have tests but are never called from production code. They are remnants of an earlier architecture. `verify.ts` also violates pure-function architecture by calling `execSync("bun test")`. These dead modules cause confusion during review (are they intentional?), contributing to reactive churn. Removing them with their test files would clean up the codebase significantly. Files: `src/skills/verify.ts`, `src/skills/prioritise.ts`, `src/skills/work.ts`, corresponding test files.

## 2. Add tests for src/utils/frontmatter.ts
**Type**: test-coverage
**Impact**: medium
**Reasoning**: `src/utils/frontmatter.ts` is used by plan detection, wiki parsing, and findings archival. It has zero test coverage despite being a foundational utility. Issues here would cascade across multiple features. Edge cases to test: CRLF line endings, empty frontmatter, missing fields, multiline values. Files: `src/utils/frontmatter.ts`, new `src/__tests__/frontmatter-utils.test.ts`.

## 3. Add tests for src/utils/housekeeping.ts
**Type**: test-coverage
**Impact**: medium
**Reasoning**: `src/utils/housekeeping.ts` contains the path-matching logic used by the auto-commit safety system. Zero test coverage. If this logic is wrong, housekeeping commits could silently include non-housekeeping changes. Files: `src/utils/housekeeping.ts`, test file.
