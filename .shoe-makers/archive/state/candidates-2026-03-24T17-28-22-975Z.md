# Candidates

## 1. Add evidence for plan-status-lifecycle invariants
**Type**: test-coverage
**Impact**: high
**Reasoning**: 4 specified-only invariants relate to plan status lifecycle (`functionality.plan-status-lifecycle`, and 3 under `spec.steer-the-elves` / `spec.maintain-a-living-spec`). The functionality is already implemented in `src/skills/assess.ts:findOpenPlans()` which correctly filters by `category: plan` and excludes `status: blocked|done`. But the invariant pipeline finds no evidence (tests) matching these claims. Adding tests that verify: (1) plans with `category: plan` are found by `findOpenPlans`, (2) plans with `status: blocked` are excluded, (3) plans with `status: done` are excluded — would satisfy these invariants.
