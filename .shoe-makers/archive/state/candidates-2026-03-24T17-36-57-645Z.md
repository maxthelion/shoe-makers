# Candidates

## 1. Remove unused evaluate() production export
**Type**: dead-code
**Impact**: low
**Reasoning**: `evaluate()` in `src/tree/evaluate.ts` is exported but only called from test files. Production code uses `evaluateWithTrace()` exclusively. The function could be unexported (keeping it as a private helper) or tests could use `evaluateWithTrace()` directly, since it returns a superset of information. Reducing the public API surface prevents accidental use of the simpler function where the trace is important.

## 2. Add missing shift summary process pattern test
**Type**: test-coverage
**Impact**: low
**Reasoning**: `computeProcessPatterns()` in `src/log/shift-summary.ts` uses the shared `REACTIVE_ACTIONS` and `PROACTIVE_ACTIONS` from `action-constants.ts`. The `shift-log-parser.test.ts` tests the parser's version of process patterns, but `shift-summary.test.ts` doesn't verify process patterns are computed correctly through the shift summary pipeline. Adding a test ensures the summary correctly reports process pattern data.

## 3. Add test for getFrontmatterField edge cases
**Type**: test-coverage
**Impact**: low
**Reasoning**: The `getFrontmatterField()` utility in `src/utils/frontmatter.ts` uses a regex that could match field names as substrings (e.g., searching for "status" might match "sub-status"). While current usage is safe, adding a test with a field name that's a substring of another field would prevent future regressions if new frontmatter fields are added.
