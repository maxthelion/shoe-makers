# Add missing config keys to integration.md config table

skill-type: doc-sync

## What needs to change

`wiki/pages/integration.md` has a config table (lines 89-96) that lists 6 config keys but is missing 2 that are defined in `src/config/load-config.ts`:

**Missing keys:**
- `max-ticks-per-shift` — default `10`, limits how many ticks run per shift
- `max-innovation-cycles` — default `3`, limits innovation ticks per shift to prevent diminishing returns

**Current table (lines 89-96):**
```
| Key | Default | Description |
|-----|---------|-------------|
| `branch-prefix` | `shoemakers` | Branch name prefix |
| `tick-interval` | `5` | Minutes between ticks (informational) |
| `wiki-dir` | `wiki` | Path to wiki directory |
| `assessment-stale-after` | `30` | Minutes before assessment needs refreshing |
| `insight-frequency` | `0.3` | Fraction of explore cycles with Wikipedia lens |
| `enabled-skills` | all | Comma-separated list of enabled skills |
```

## Exact change

Add two rows to the config table:

```
| `max-ticks-per-shift` | `10` | Maximum ticks per shift |
| `max-innovation-cycles` | `3` | Maximum innovation ticks per shift |
```

## What NOT to change

- Do not modify src/, tests, or other wiki pages
- Do not modify .shoe-makers/invariants.md
- Do not reorder or modify existing table rows

## Decision Rationale

Chose this over archive tests (#2) and architecture docs (#3) because it directly affects users setting up shoe-makers — they can't configure these keys if they don't know they exist. The known-issues entry (#4) is also useful but less impactful since the npm registry issue is environment-specific.
