# Candidates

## 1. Add roundtrip test for init skill templates
**Type**: test-coverage
**Impact**: medium
**Reasoning**: `src/init-skill-templates-docs.ts` and `src/init-skill-templates-quality.ts` define skill templates that `shoemakers init` writes to disk. The existing `init-templates.test.ts` checks that templates exist and have frontmatter, but doesn't verify the templates can be parsed by `parseSkillFile()` from the registry. A roundtrip test (generate template → parse with registry) would catch format regressions. This is especially important because the skill registry validates frontmatter fields and throws on missing required fields.

## 2. Remove unused evaluate() production export
**Type**: dead-code
**Impact**: low
**Reasoning**: `evaluate()` in `src/tree/evaluate.ts` is exported but only used in test files. Production code always calls `evaluateWithTrace()`. Removing the export (or making it unexported) reduces the public API surface. Tests that use `evaluate()` can switch to destructuring the result of `evaluateWithTrace()`.

## 3. Add process pattern detection to shift summary integration
**Type**: test-coverage
**Impact**: low
**Reasoning**: The `computeProcessPatterns()` function in `src/log/shift-summary.ts` uses the shared `REACTIVE_ACTIONS` and `PROACTIVE_ACTIONS` constants, but the shift summary integration tests don't verify that the process patterns field is populated correctly when the shift summary includes reactive actions. Adding a test would ensure the constants consolidation doesn't cause regressions in shift summary output.
