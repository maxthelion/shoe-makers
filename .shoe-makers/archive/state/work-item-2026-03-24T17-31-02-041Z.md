# Fix plan-status invariant evidence regression

skill-type: bug-fix

## Problem

The frontmatter parsing refactor (commit 333694f) changed `src/skills/assess.ts` from:
```ts
if (frontmatter.match(/status:\s*(blocked|done)/)) continue;
```
to:
```ts
const status = getFrontmatterField(parsed.frontmatter, "status");
if (status === "blocked" || status === "done") continue;
```

This broke 4 invariant evidence patterns in `.shoe-makers/claim-evidence.yaml` that relied on the old regex strings being present in the source. The evidence patterns look for:
- `"status:\\s*(blocked|done)"` — old regex literal, no longer present
- `"status: blocked"` — not present in new code
- `"status: done"` — not present in new code

Similarly, `"category:\\s*plan"` is no longer a literal in the source after the refactor.

## What to do

1. Update `.shoe-makers/claim-evidence.yaml` to match the new code patterns:

   For `functionality.plan-status-lifecycle`:
   - Source evidence group 2: add `"blocked"` and `"done"` as alternatives (these strings ARE present in the new code as `=== "blocked"` / `=== "done"`)

   For `spec.steer-the-elves.write-a-plan-page-in-the-wiki-category-plan-and-elves-work-t`:
   - Source evidence group 2: add `'"plan"'` as alternative (present in `!== "plan"`)

   For `spec.steer-the-elves.mark-a-plan-as-status-blocked-or-status-done-to-control-what`:
   - Source evidence group 1: add `"blocked"` and `"done"` as alternatives

   For `spec.maintain-a-living-spec.plans-describe-what-the-system-should-become-deltas-that-are`:
   - Source evidence group 2: add `'"plan"'` as alternative

2. Run `bun test` to confirm invariant tests pass

3. Run `bun run setup` to verify specified-only count drops back to 0

## What NOT to change

- Do not modify `.shoe-makers/invariants.md`
- Do not change the source code (the refactored code is correct)
- Do not change unrelated evidence patterns

## Decision Rationale

This is a direct regression fix. The frontmatter refactor was correct but inadvertently broke evidence matching because the evidence patterns were based on the old code's string literals. The evidence patterns need updating to match the refactored code.
