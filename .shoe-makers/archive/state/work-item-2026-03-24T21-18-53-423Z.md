skill-type: doc-sync

## What needs to change

### New entry content

```markdown
## Typecheck and health score show as "unknown" in cloud environments

**Symptom**: `bun run setup` reports `Uncertainties: typecheckPass (missing type definitions (bun-types)), healthScore (octoclean not installed)`. The behaviour tree trace shows `✗ tests-failing (2 unknowns: typecheckPass, healthScore)`.

**Cause**: When npm registry access is blocked (see above), `bun-types` and `octoclean` can't install. The typecheck (`tsc --noEmit`) fails because it can't find type definitions for Bun APIs. The health scan returns null because octoclean isn't available.

**Impact**: The `tests-failing` tree condition treats unknowns cautiously — it does NOT trigger when typecheckPass or healthScore are unknown (only when they're explicitly `false`). This means the tree correctly skips fix-tests and proceeds to the next condition. The unknowns are logged but don't block work.

**Fix**: No action needed. This is expected in restricted cloud environments. The system works correctly despite the uncertainties. If you need type checking, ensure `bun install` can access the npm registry.
```
